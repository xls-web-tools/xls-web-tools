# xls-web-tools

xls-web-tools は、Web 上の情報取得・整形・検証を支援する Excel マクロ ツール群です。
各ツールは `.xlsm` と `modules` を同じフォルダーに置き、VBA ソースを `modules` で管理します。

## フォルダー構成

- `common_modules_repo`: xls-web-tools に配布された共通モジュール。変更時は `xls-common-devtools/CommonModules/modules` を原本として扱います。
- `Web情報取得`: Web 上の情報を取得し、Excel で扱える形へ整形・確認するツール。
- `Web情報取得/modules`: Web 情報取得ツール固有の VBA ソースとユニットテストを管理する場所。
- `docs/agents`: Agent skills が参照する Issue tracker と domain docs の運用ルール。
- `docs/adr`: Web ツール群の architecture decision records。

## Language

**Web 関連ツール**:
Web ページ、HTTP API、公開データなどの取得・整形・確認を Excel から支援するツール群。
_Avoid_: 一般的な Web サイトや Web アプリ全般

**Web情報取得**:
Web 上の情報を取得し、Excel ブック上で利用・確認できる形式へ変換するツール。
_Avoid_: xls-web-tools 内の全ツール

**取得対象**:
Web情報取得が読み取り対象とする URL、HTTP API、HTML、JSON、CSV などの外部情報源。
_Avoid_: 取得後に Excel 上へ書き込まれた結果

**取得結果**:
取得対象から読み取って、Excel 上で確認または後続処理に使える形へ整えたデータ。
_Avoid_: 生のレスポンスだけ

**ブラウザ取得**:
Edge WebDriver で Microsoft Edge を操作し、認証・リダイレクト・JavaScript 実行後のページ状態から取得結果を作る方式。
_Avoid_: Excel VBA から HTTP リクエストだけで取得対象を読む方式

**専用ブラウザプロファイル**:
Web情報取得がブラウザ取得で使う、通常利用の Edge プロファイルとは分離した Cookie、認証状態、サイト設定の保持領域。
_Avoid_: 利用者が普段使う Edge プロファイル

**認証遷移**:
取得対象へ到達する前に発生する、ログイン確認ページ、認証成功ページ、リダイレクト、Cookie 設定などの一連のページ遷移。
_Avoid_: Web情報取得が保存・再利用する認証情報

**認証後の起点ページ**:
認証遷移が完了したことを確認し、一覧画面へ進むための起点となるページ。一覧画面と同一ページになる場合もあります。
_Avoid_: 必ず表示されるトップページ

**一覧画面**:
取得対象の候補となる一覧項目が複数並ぶ画面。
_Avoid_: Web情報取得の設定一覧シート

**一覧項目**:
一覧画面上で、1 つの詳細ページへ遷移する単位。
_Avoid_: ワークシート上の出力行

**収集準備進捗**:
Web情報取得で、WebDriver session 開始から一覧画面到達までの準備がどこまで進んだかを示す状態。一覧項目数が判明する前の固定ステップ進捗として扱います。
_Avoid_: 一覧項目の処理完了数、詳細ページを開いた件数、出力行を作成した件数

**収集進捗**:
Web情報取得の本番収集がどこまで進んだかを示す状態。一覧項目の処理完了数を基準にし、既存行扱いや出力対象条件によって詳細ページを開かずにその一覧項目の扱いが決まった場合も処理完了に含めます。
_Avoid_: 収集準備進捗、詳細ページを開いた件数だけ、出力行を作成した件数だけ

**一覧上の対象ID**:
一覧画面上の一覧項目に表示される、遷移前に候補を識別するための対象ID相当の値。詳細ページ上の対象IDと一致することを期待します。
_Avoid_: 画面遷移操作だけに使う内部識別値、出力行の正となる対象ID

**画面遷移操作**:
取得対象サイトが提供する onClick の JavaScript 呼び出しなど、利用者操作と同等の方法で画面を移動する操作。
_Avoid_: URL を組み立てて直接開く画面移動

**画面遷移操作のテキスト絞り込み**:
画面遷移操作で locator だけでは対象要素を一意に決められない場合に、候補要素の正規化した表示テキストの完全一致で操作対象を特定する補助条件。詳細遷移操作だけでなく、一覧遷移、一覧復帰、次ページ遷移にも使える条件として扱います。
_Avoid_: 詳細遷移操作だけの特別条件、出力値抽出の条件

**遷移後待機 selector**:
画面遷移操作を実行した後に、遷移先画面へ到達したことを判定する selector。操作対象 selector とは別の条件であり、一覧画面到達も含めて、一覧遷移、詳細遷移、一覧復帰、次ページ遷移ごとに対応します。
_Avoid_: 操作対象 selector、次ページ操作可否の事前判定 selector、抽出列 selector

**詳細遷移操作**:
一覧項目から詳細ページへ入るために、取得対象サイトのクリック操作と同等の動きを WebDriver で再現する操作。
_Avoid_: 詳細ページ URL を組み立てて直接開く操作

**詳細ページ**:
一覧項目から遷移する、1 件分の情報を表示するページ。
_Avoid_: 一覧画面、ログイン直後のトップページ

**ダウンロードリンク**:
詳細ページ上で、対象IDに紐づくファイルのブラウザダウンロードを開始するリンクまたはボタン。Web情報取得は認証済みの Edge session 上で利用者操作と同等に実行します。
_Avoid_: VBA の HTTP リクエストで直接取得するファイル URL

**ダウンロード対象ファイル**:
詳細ページ上のダウンロードリンクから取得する、対象IDに紐づくファイル。1 つの詳細ページに対して存在しない場合もあり、存在する場合は 1 つとして扱います。ZIP ファイルの場合は、最終出力では ZIP 自体ではなく展開結果を扱います。
_Avoid_: 詳細ページから抽出したセル出力用の値、複数添付ファイルの集合

**ダウンロード保存先ルート**:
Web情報取得がダウンロードしたファイルを対象ID別のフォルダーへ振り分けるための親フォルダー。各対象の保存先フォルダーは、この配下に対象ID名で作られます。
_Avoid_: 対象IDごとの最終保存フォルダー、ブラウザプロファイルの既定ダウンロードフォルダー

**ダウンロード状態**:
出力行に対応するダウンロード対象ファイルの取得結果。DOWNLOADED、NO_FILE、SKIPPED、ERROR のいずれかで表します。SKIPPED は、同名の既存ファイルを保護するために今回のダウンロードを実行しなかった状態です。
_Avoid_: 詳細ページ全体の取得状態、保存済みファイルパス

**一覧復帰リンク**:
詳細ページから一覧画面へ戻るために、取得対象サイトが提供する所定のリンク。
_Avoid_: ブラウザーの戻るボタン、詳細ページ URL の直接再利用

**次ページ操作**:
一覧画面で現在ページの次の一覧項目群へ進むために、取得対象サイトが提供するリンクやボタンを利用者操作と同等に実行する操作。
_Avoid_: ブラウザーの進むボタン、一覧ページ URL を直接組み立てたページ移動

**対象ID**:
詳細ページ上に表示される、出力行の対象を業務上識別する値。詳細ページ上の表示値を正とします。
_Avoid_: 画面遷移操作だけに使う内部識別値、一覧画面上だけで確認した一覧上の対象ID、画面到達判定だけに使う selector

**出力行**:
1 つの詳細ページから抽出した、ワークシート上の 1 行分の取得結果。
_Avoid_: 詳細ページそのもの

**抽出列**:
詳細ページから値を読み取る対象として定義された列。出力対象条件の判定に使うためだけに抽出し、出力行へ記録しない場合もあります。
_Avoid_: output シートのヘッダーで選ばれた出力列、固定管理列

**派生列**:
他の抽出列または派生列の値から計算される列。詳細ページから直接読み取る列ではなく、抽出結果に基づいて出力行や出力対象条件で使う値を表します。
_Avoid_: DOM selector で直接値を読む抽出列、output シートのヘッダーで選ばれた出力列

**列参照記法**:
OutputColumnName を [列名] の形式で参照し、比較、論理、文字列リテラル、IF により出力対象条件や派生列の値を記述する settings 上の記法。
_Avoid_: Excel のセル数式、VBA 関数、DOM selector

**出力列**:
抽出列または派生列のうち、output シートの固定管理列より右側の非空ヘッダーで選ばれ、出力行へ記録する列。
_Avoid_: 出力対象条件の判定だけに使う抽出列または派生列、固定管理列

**出力値型変換方針**:
抽出列または派生列の値を出力行へ記録するとき、文字列として扱うか、ワークシート上の値として自動判定するかを表す詳細ページ列定義ごとの方針。
_Avoid_: 出力列そのもの、値式の戻り値型、固定管理列の書き込み方針

**取得状態**:
出力行が詳細ページ単位の取得に成功したかを示す固定管理列。OK または ERROR のいずれかで表します。
_Avoid_: ダウンロード対象ファイルの取得結果、出力対象条件の一致結果

**取得エラー行**:
詳細ページ単位の取得失敗として、固定管理列の 取得状態 に ERROR を記録した出力行。
_Avoid_: ダウンロード状態=ERROR だが 取得状態=OK の出力行、出力対象外

**出力対象条件**:
詳細ページから取得した値に基づき、出力行を作る対象かどうかを判定する条件。
_Avoid_: 取得設定全体、詳細ページ列定義、既存行扱い

**出力対象外**:
詳細ページから取得した値が出力対象条件に一致せず、output シートへ新規行または ERROR 行として記録しない扱い。既存の出力行がある場合も削除しません。
_Avoid_: 取得失敗、必須項目欠落、既存行スキップ

**出力シート不整合**:
output シートの既存ヘッダーや固定管理列の値が、現在の取得設定に基づく再実行を安全に続けられない状態。
_Avoid_: 詳細ページ単位の取得エラー、出力対象外、settings エラー

**既存正常行スキップモード**:
取得状態=OK の既存の出力行を再取得せず、未登録または 取得状態=ERROR の出力行を取得する再実行時の扱い。
_Avoid_: 取得状態=ERROR の既存行までスキップする収集、全更新モード

**全更新モード**:
既存の取得状態に関係なく、一覧項目から詳細ページを再取得し、対応する出力行を更新する再実行時の扱い。
_Avoid_: 新規行だけを追加する収集、既存 OK 行をスキップする収集

**取得設定**:
取得先、取得方法、認証やヘッダー、出力先シートなど、Web 情報取得の動作を決める設定。
_Avoid_: ユーザーが一時的に入力する検索語だけ

**共通モジュール**:
`common_modules_repo` から各ツールの `modules` へ配布される VBA 基盤。変更時は `xls-common-devtools/CommonModules/modules` を原本として扱います。
_Avoid_: xls-web-tools 固有モジュール

## Development Notes

- Web ツール固有の業務処理は対象ツールの `modules` に置きます。
- `common_modules_repo` に含まれるファイルを直接原本として編集しません。共通モジュール変更は `xls-common-devtools/CommonModules/modules` から始めます。
- 取得対象、取得設定、シート名、列番号、業務文言を共通モジュールへ混入させません。
- 外部 Web へのアクセスを伴う処理は、通常の単体テストではテストダブルまたは固定テストデータへ切り出し、ネットワーク状態に依存しない検証を優先します。
- Excel への import/export や UnitTestMain 実行は Excel COM と VBIDE アクセスを使うため、実行前に対象 `.xlsm` が閉じていることを確認します。

## Example Dialogue

開発者: `取得結果` は HTTP の生レスポンス文字列だけを指しますか？
ドメイン専門家: いいえ。取得対象から読み取り、Excel 上で確認や後続処理に使える形へ整えたデータを指します。
